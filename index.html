FC_Polydart

Point of Sale/Enterprise Suite Web App, built with Polymer and Dart

View project on GitHub

Welcome to our page!

This is an ambitious project, attempting to build a med-large scale enterprise suite and point of sale solution using relatively new technologies, namely Dart and Polymer. Dart is a Java/Javascript like language which can be run in vm or compiled to JS. Polymer is Google's take on Web Components. Our backend will be pure Dart/MySQL, leveraging several other open source Dart projects along the way. Our frontend will use Polymer.dart, compiled to JS for browser compatibility.

Goals

Some goals we have, in no particular order:

  • A NoSQL session store for tracking active users.
  • An RPC api supporting the Discovery Document format, so other frontends can conceivably be written.
  • A Polymer front end implementing the Material Design spec.
  • Encrypted traffic using RSA.
  • A powerful local api, to allow for non-web app front ends.
  • Full stack Dart (with frontend compiled to JS and vulcanized)
  • Inventory Tracking
  • Multiple location tracking
  • Customer Relationship Management
  • Invoicing
  • HR Management
  • Event scheduling
  • Shipment tracking
  • CC Processing
  • Bitcoin?
  • Touch screen friendly interface (frontend)
  • Android app (someday, Discovery Document format should help with porting)
  • Offline support where possible (This will be necessarily limited)
  • Free puppies for early adopters
  • Barcode label generation
  • A secretary to keep goal lists from getting out of hand

Leveraged Projects

There are several external projects we are using in our server-side code. Some are fresher than others, but we hope to be able to contribute some changes back, bringing the stale projects we're using back to life. The projects we are working with server-side include: RPC, Cipher (Though soon to migrate to PointyCastle, which is actively maintained), AvocadORM, uuid, redis_client, validator, and a few more, indirectly. Some of these projects have been forked into my repo list for the purposes of rooting out bugs and making it all play nicely together. Once my main goals have been met (a functional app, for one), I plan to go back, clean up any changes I made, and open pull requests back where appropriate.
